# A walking diary (걸음수 관리)

브라우저에서 동작하는 걸음수·누적 기록 앱입니다. 챌린지 **시작일·종료일·목표 걸음수**는 **설정**에서 바꿀 수 있으며, 기본값은 2026년 3월 11일 ~ 2027년 3월 10일, 목표 4,000,000보입니다.

## 주요 기능

### 입력·기준 날짜

- **기준 날짜**: 기본은 **어제**. 날짜를 누르면 달력으로 변경 (설정한 챌린지 기간 안에서만 선택).
- **걸음 입력 방식** (라디오, 선택은 `walkingDiaryStepsEntryMode`로 기억됨)
  - **기준날짜까지 누적된 걸음수**: 입력값을 그날의 **누적 걸음**으로 저장. 이전 날짜 기록 여부와 관계없이 저장 가능.
  - **기준날짜 걸음수**: 그날 **하루 걸음**으로 보고, 저장 시 **전날(달력)까지의 누적 + 입력값**을 누적 걸음으로 저장.  
    단, **전날이 챌린지 기간 안에 포함될 때**는 전날짜에 이미 저장된 기록이 있어야 하며, 없으면 `이전 날짜 걸음수 정보를 먼저 입력하세요.` 안내 후 저장하지 않습니다. (챌린지 **시작일**은 전날이 기간 밖이므로 이 검사를 적용하지 않고, 전날 누적 0부터 더해 저장할 수 있습니다.)
- 입력창에는 방식에 맞게 누적값 또는 (저장된 누적 − 전날 누적) 하루 걸음이 표시됩니다.
- **×**: 입력창만 비움 (저장된 기록은 그대로).
- **확인**: 위 규칙에 따라 저장.

### 메인 통계 카드

- 제목 **걸음수 통계** 옆에 현재 목표를 `[목표 4,000,000보]` 형식으로 표시합니다. **숫자 부분만** 밝은 노란색으로 강조됩니다.
- 기준일, 걸어온 날, 걸어온 걸음, 평균 하루 걸음, 남은 날, 남은 걸음수, 평균 걸어야 될 걸음수를 표시합니다.

### 통계 팝업 (**통계** 버튼)

- **하나의 팝업**에 두 그래프를 세로로 배치합니다.
- **최근 30일 누적 걸음수** (위)
  - 가장 최근 기록일부터 역으로 30일, 날짜별 **그날까지의 누적** 추이(꺾은선).
  - **목표 누적** 꺾은선(챌린지 기간 대비 선형 목표)과 비교.
  - 가로축 날짜: **오늘(로컬)** 인 날과, 오늘과의 일수 차이가 **7의 배수**인 날만 표시.
  - 목표 누적 설명 문구는 그래프 **상단**에 표시.
- **최근 7일 걸음수** (아래)
  - 일별 추정 걸음을 **막대**로 표시.
  - 막대를 **누르거나 클릭**하면 해당 날의 **날짜·걸음수**가 풍선 형태로 표시됩니다. 막대 밖을 누르면 사라지며, 팝업을 닫을 때도 함께 닫힙니다.
  - **일주일 평균**(빨간 점선)과 **평균 걸어야 될 걸음**(초록 점선)을 그래프에 표시하고, 같은 문구를 **그래프 아래**에도 큰 글씨로 표시.
- 7일 막대 그래프 **아래**에는 **SVG**로 걷는 사람 애니메이션과, 배경에서 **건물·나무가 서로 다른 속도로 흐르는** 패럴랙스 효과가 있습니다. `prefers-reduced-motion: reduce`가 켜진 환경에서는 이 움직임이 비활성화됩니다.
- 모바일(좁은 화면)에서는 통계 팝업이 **전체 화면**에 가깝게 열립니다.

### 설정 (**설정** 버튼)

- **시작일·종료일·목표 걸음수** 표시.
- **초기화**: 입력을 활성화하고 시작일을 **오늘**로 두며 **저장** 버튼을 켭니다.
- **저장**: 현재 히스토리와 설정을 `walkingHistory_old`에 백업한 뒤, 새 기간·목표를 적용하고 걸음 히스토리를 비웁니다.
- **복구**: 백업이 있을 때만 활성화. 히스토리와 (백업에 포함된 경우) 설정을 되돌립니다.

### 헤더 버튼 배치

- **통계**: 밝은 반투명 스타일(헤더 위에서 강조).
- **설정**: 회색 톤 버튼.

## 사용 방법

1. `index.html`을 브라우저에서 엽니다.
2. **기준 날짜**를 필요하면 바꿉니다.
3. **누적** 또는 **기준일 걸음** 방식을 고른 뒤 값을 입력하고 **확인**으로 저장합니다. (기준일 걸음은 전날 기록이 필요할 수 있음.)
4. 아래 **걸음수 통계** 카드에서 진행 상황과 **목표 걸음** 문구를 확인합니다.
5. **통계**로 30일 누적·최근 7일 그래프를 보고, 7일 막대에서 값을 확인하거나 아래 장식 애니메이션을 봅니다.
6. 챌린지 기간이나 목표를 바꿀 때는 **설정**을 사용합니다 (저장 시 기존 기록은 백업됩니다).

## 데이터 저장 (localStorage)

| 키 | 설명 |
| --- | --- |
| `walkingHistory` | 날짜별 누적 걸음 `{ "YYYY-MM-DD": number }` |
| `walkingDiarySettings` | `{ startDateKey, endDateKey, targetSteps }` JSON |
| `walkingDiaryStepsEntryMode` | `cumulative` 또는 `daily` — 걸음 입력 라디오 선택 |
| `walkingHistory_old` | 설정 **저장** 직전의 히스토리·설정 백업(JSON). **복구** 시 되돌리고 삭제 |

## 파일 구조

- `index.html` — 마크업, 통계·설정 모달, 걷기 SVG(패럴랙스 배경)
- `styles.css` — 레이아웃·반응형·모바일 풀 통계 팝업·애니메이션 등
- `script.js` — 저장/통계/그래프/막대 툴팁/설정·백업 로직

## 기술 스택

- HTML / CSS / JavaScript (Vanilla)
- Canvas 2D (통계 그래프)
- SVG + CSS 애니메이션 (통계 팝업 하단 장식)
- Web Storage API (localStorage)
